# Farcaster Mini App Test Harness

A reusable test harness for Farcaster Mini Apps. The harness lives at:

```
https://github.com/creativelayer/farcaster-test-harness
```

## Quick Setup

In any Mini App project, clone the harness as a sibling directory:

```bash
cd ~/projects
git clone https://github.com/creativelayer/farcaster-test-harness
cd farcaster-test-harness
pnpm install
npx playwright install chromium
```

Serve the host emulator on port 4000:

```bash
pnpm serve
```

Run tests pointing at your app on port 3000:

```bash
pnpm test
```

## Critical: The Comlink Wire Protocol

The Farcaster Mini App SDK (`@farcaster/miniapp-sdk`) uses **Comlink** internally for all postMessage communication. This is not documented anywhere obvious. All response values must be wrapped in Comlink's wire envelope:

```json
{ "type": "RAW", "value": <actual data> }
```

Without this wrapper, `fromWireValue()` returns `undefined` and `sdk.context` silently resolves to `null`.

### Full Protocol Reference

**App → Host (GET property, e.g. `sdk.context`):**
```json
{ "id": "<correlation-id>", "type": "GET", "path": ["context"] }
```

**Host → App (GET response):**
```json
{ "id": "<same-correlation-id>", "type": "RAW", "value": <contextObject> }
```

**App → Host (APPLY method, e.g. `sdk.actions.ready()`):**
```json
{
  "id": "<correlation-id>",
  "type": "APPLY",
  "path": ["ready"],
  "argumentList": [{ "type": "RAW", "value": {} }]
}
```

**Host → App (APPLY response):**
```json
{ "id": "<same-correlation-id>", "type": "RAW", "value": null }
```

The `id` is a correlation ID generated by comlink — always echo it back exactly.

## Context Fixture Schema

The context object sent in response to `GET context` must include these fields. The SDK returns `null` if required fields are missing:

```javascript
{
  user: {
    fid: 3621,                             // number, required
    username: 'testuser',                  // string, required
    displayName: 'Test User',              // string, required
    pfpUrl: 'https://example.com/pfp.png', // string, required
  },
  client: {
    clientFid: 9152,                       // number, required
    added: false,                          // boolean, required
    safeAreaInsets: {                       // object, required (all fields required)
      top: 0, bottom: 0, left: 0, right: 0
    },
  },
  location: {
    type: 'launcher',                      // 'launcher' | 'cast_embed' | 'notification'
  },
}
```

### Fixture Variants

The harness includes three built-in fixtures selectable via `?fixture=` query param:

**launcher** (default) — standard app launch:
```javascript
{ user: { fid: 3621, ... }, client: { added: false, safeAreaInsets: { top: 0, bottom: 0, left: 0, right: 0 } }, location: { type: 'launcher' } }
```

**cast_embed** — launched from a cast:
```javascript
{ user: { fid: 12345, ... }, client: { added: false, safeAreaInsets: { top: 44, bottom: 34, left: 0, right: 0 } }, location: { type: 'cast_embed', cast: { hash: '0xabc123', author: { fid: 99, username: 'caster' }, text: 'Check out this app!' } } }
```

**notification** — launched from a notification:
```javascript
{ user: { fid: 12345, ... }, client: { added: true, notificationDetails: { url: '...', token: '...' }, safeAreaInsets: { top: 44, bottom: 34, left: 0, right: 0 } }, location: { type: 'notification', notification: { notificationId: 'notif-1', title: 'Your turn!', body: 'Come back and play' } } }
```

## Host Emulator Message Handler

The core message listener in `host.html`. Use this exact pattern — do not improvise the protocol:

```javascript
window.addEventListener('message', (event) => {
  if (event.source !== frame.contentWindow) return

  const { id, type, path, argumentList } = event.data || {}
  const pathKey = (path || []).join('.')

  // GET context — the SDK sends { type: 'GET', path: ['context'] }
  if (pathKey === 'context') {
    frame.contentWindow.postMessage({ id, type: 'RAW', value: context }, '*')
    return
  }

  if (type === 'APPLY') {
    if (pathKey === 'ready') {
      frame.contentWindow.postMessage({ id, type: 'RAW', value: null }, '*')
      statusEl.textContent = 'READY'
      statusEl.className = 'ready'
      return
    }

    if (pathKey === 'addMiniApp') {
      frame.contentWindow.postMessage({ id, type: 'RAW', value: { added: true } }, '*')
      return
    }

    if (pathKey === 'signIn') {
      frame.contentWindow.postMessage({
        id, type: 'RAW',
        value: { message: 'mock-siwf-message', signature: '0xmocksignature123' }
      }, '*')
      return
    }

    // catch-all for unknown APPLY paths
    frame.contentWindow.postMessage({ id, type: 'RAW', value: null }, '*')
    return
  }
})
```

## Playwright Test Pattern

Standard test asserting full SDK handshake and FID display:

```typescript
import { test, expect } from '@playwright/test'

test('mini app receives FC context and calls ready', async ({ page }) => {
  page.on('console', msg => console.log('HOST:', msg.text()))

  await page.goto('http://localhost:4000/host.html?url=http://localhost:3000&fixture=launcher')

  // Assert SDK handshake completes
  await expect(page.locator('#status')).toHaveText('READY', { timeout: 10000 })

  // Assert FID rendered in app iframe
  const appFrame = page.frameLocator('iframe#miniapp-frame')
  await expect(appFrame.locator('[data-testid="fid-display"]')).toHaveText('fid:3621', { timeout: 5000 })
})
```

The FID `3621` matches the launcher fixture. Update if you change the fixture's `user.fid`.

## App-Side SDK Integration

Correct pattern for calling the SDK in a Next.js client component:

```typescript
'use client'
import { useEffect, useState } from 'react'
import { sdk } from '@farcaster/miniapp-sdk'

export default function Home() {
  const [fid, setFid] = useState<number | null>(null)

  useEffect(() => {
    const init = async () => {
      await sdk.actions.ready()           // call ready() first
      const context = await sdk.context   // then read context
      if (context) setFid(context.user.fid)
    }
    init()
  }, [])

  return (
    <div data-testid="fid-display">
      {fid ? `fid:${fid}` : 'no fc context'}
    </div>
  )
}
```

**Important:** Call `ready()` before reading `sdk.context`. The host sets status to READY on the `ready()` call, and context is available independently, but calling ready first ensures the handshake is initiated.

## Ralph PRD Integration

When building Mini Apps with Ralph, include these stories:

**US-xxx — FC Host Emulator Setup:**
> Clone `https://github.com/creativelayer/farcaster-test-harness` as a sibling directory to the app. Install dependencies with `pnpm install`. The harness serves on port 4000 via `pnpm serve`. Do not reimplement the host — use the repo as-is.

**US-xxx — Playwright Integration Test:**
> Write a Playwright test in the harness `tests/` directory. Load `http://localhost:4000/host.html?url=http://localhost:3000&fixture=launcher`. Assert `#status` becomes `READY`. Assert `[data-testid="fid-display"]` in the app iframe shows `fid:3621`. Run from the harness directory with `pnpm test`.

## Troubleshooting

**`sdk.context` resolves to null** — Response is missing the `{ type: "RAW", value: ... }` Comlink wrapper. Every postMessage response from host to app must use this envelope.

**Status stays WAITING** — The `APPLY ready` path is not being matched. Log `event.data` raw and check the `path` array. It will be `["ready"]` not `["actions", "ready"]` — comlink flattens the proxy chain.

**FID displays but wrong value** — Fixture `user.fid` doesn't match test assertion. The launcher fixture uses FID `3621`; cast_embed and notification use `12345`.

**iframe content not accessible in Playwright** — Use `page.frameLocator('iframe#miniapp-frame')` not `page.frames()`. The iframe must have the `id` attribute set in `host.html`.

**`serve` redirects .html URLs** — The `serve` package does clean URL redirects (`host.html` → `host`). Both work; Playwright follows redirects automatically.
